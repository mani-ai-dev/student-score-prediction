## 🧠 Overview

This project demonstrates a classic supervised machine learning pipeline — from **data creation** to **model training**, **evaluation**, and **deployment** via a Streamlit web app. It explores how many hours a student studies and predicts their exam score using two regression approaches:

- **Linear Regression** — fits a straight line to the data
- **Polynomial Regression (Degree 2)** — fits a curved line to capture non-linear patterns

The app allows users to interactively select study hours and a model type to get a real-time predicted score, along with a side-by-side metrics dashboard and regression plot.

|--------|-------------|

- **Format:** CSV (`student_scores.csv`) generated automatically
- **Split:** 80% Training / 20% Testing (`random_state=42`)

## ⚙️ Installation

### Option 1: Run on Google Colab (Recommended)

1. Upload `student_score_prediction.ipynb` to [Google Colab](https://colab.research.google.com/)
2. Run all cells in order
3. When prompted, enter your [ngrok authtoken](https://dashboard.ngrok.com/get-started/your-authtoken)
4. Click the generated public URL to access the Streamlit app

### Option 2: Run Locally

**Prerequisites:** Python 3.7+

**Step 1: Clone the repository**
```bash
git clone https://github.com/your-username/student-score-predictor.git
cd student-score-predictor
```

**Step 2: Install dependencies**
```bash

```

**Step 3: Run the notebook** (to generate `app.py` and `student_scores.csv`)
```bash
jupyter notebook student_score_prediction.ipynb
```

**Step 4: Launch the Streamlit app**
```bash
streamlit run app.py
```

**Step 5: Open in browser**
```
http://localhost:8501
```

## 🚀 Usage

Once the Streamlit app is running:

1. **Adjust the slider** to select your desired study hours (1.0 – 10.0)
2. **Select a model** using the radio button:
   - `Linear` — uses Linear Regression
   - `Polynomial` — uses Polynomial Regression (degree 2)
3. **View the predicted score** displayed instantly below the controls
4. **Check the metrics table** for R², MAE, and RMSE comparison between models
5. **Examine the regression plot** showing actual data points with both model fit lines

## 🤖 Model Details

### Linear Regression

Fits a straight line: `Score = m × Hours + b`

- Trained using `sklearn.linear_model.LinearRegression`
- Simple, interpretable, performs well on linearly distributed data

### Polynomial Regression (Degree 2)

Transforms features as: `[Hours, Hours², 1]`, then applies Linear Regression.

- Transformation via `sklearn.preprocessing.PolynomialFeatures(degree=2)`
- Better captures non-linear growth patterns in the data
- More flexible but risks overfitting on very small datasets

---

## 📊 Performance Metrics

Three standard regression metrics are used to evaluate both models:

| Metric                             | Description                                                                                |
|------------------------------------|--------------------------------------------------------------------------------------------|
| **R² (R-squared)**                 | Proportion of variance explained by the model. Closer to 1.0 is better.                    |
| **MAE (Mean Absolute Error)**      | Average absolute difference between predicted and actual scores. Lower is better.          |
| **RMSE (Root Mean Squared Error)** | Square root of average squared differences. Penalizes larger errors more. Lower is better. |

> Expected performance: Both models achieve **R² > 0.97** on this dataset due to its strong linear correlation.

## 💡 Results & Insights

- Study hours and exam scores show a **strong positive correlation**
- Both models perform well, with Polynomial Regression having a **slightly better fit** on training data
- Linear Regression is preferred for **simplicity and generalizability**
- The dataset is small (16 records), so both models may perform similarly on unseen data
